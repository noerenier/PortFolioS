Afin de mieux appréhender la gestion d'un parc informatique ainsi que la gestion des incidents au sein d'une entreprise, nous avons déployé GLPI et GESTSUP. Cette démarche vise à nous familiariser avec la création la gestion des incidents et d'un parc informatique.

Catégorie: Gestion de configuration
Compétence: Répondre aux accidents et aux demandes d'assistance et d'évolution

Installation GLPI :

Assistant d’installation : Pour débuter l’installation, faire pointer le navigateur web sur le dossier racine de GLPI : https://127.0.0.1/ Lorsque GLPI n’est pas installé, un processus d’installation étape par étape commence.

Choix de la langue (Select your language) :

La première étape permet de choisir la langue dans laquelle se déroulera l’installation. Sélectionner la langue souhaitée, puis valider.

Licence :

L’utilisation de GLPI est soumise à l’acceptation de la licence d’utilisation GNU. Une fois la licence lue et acceptée, valider le formulaire. Sans acceptation des termes de la licence, il est impossible d’accéder aux étapes suivantes.

Installation / mise à jour :

Cet écran permet de choisir entre installer GLPI ou effectuer une mise à jour. Puis cliquer sur Installation.

Vérification de l’environnement :

Cette étape va vérifier que le système satisfait les pré-requis. Si ce n’est pas le cas, il est impossible de continuer et un message d’erreur explicite indique les actions à réaliser avant d’essayer à nouveau. Certains pré-requis sont optionnels, il sera alors possible de valider le formulaire même s’ils ne sont pas respectés.

Connexion à la base de données :

Les paramètres de connexion à la base de données sont demandés.
Database connection parameters
Serveur MySQL : saisir le chemin de MySQL, par exemple localhost, ou mysql.domaine.tld ;
Utilisateur MySQL : saisir le nom d’utilisateur ayant le droit de se connecter au serveur MySQL ;
Mot de passe MySQL : saisir le mot de passe associé à l’utilisateur. Une fois que ces champs sont correctement remplis, valider le formulaire. Une première connexion à la base est alors établie. Si les paramètres sont invalides, un message d’erreur est affiché, il faut modifier les paramètres de connexion et essayer à nouveau.

Choix de la base de données :

Une fois la connexion au serveur de base de données établie, il faut créer ou choisir la base de données souhaitée pour GLPI et l’initialiser.

Installation GESTSUP :

Téléchargez la dernière version du package de l'application depuis la section Téléchargements.
Extraire les fichiers à la racine de votre serveur web, exemple : "c:\wamp64\www".
Lancez WAMP, puis allez sur votre navigateur et accédez à l'URL suivante : http://127.0.0.1/install/.
Dans l'interface d'installation : Précisez les paramètres de la base de données (S'il s'agit de WAMP laissez tout par défaut, ne pas mettre de mot de passe).
Dans l'interface d'installation : Activez les modules PHP nécessaires, via le menu WAMP présent dans la barre des tâches, un redémarrage de WAMP sera nécessaire.
Une fois l'installation finie aller sur l'URL http://127.0.0.1 pour accéder à l'application.
Après être connecté à l'application, n'oubliez pas de changer les mots de passe par défaut des utilisateurs initiaux. (admin, user...)
Une fois que vous avez validé le bon fonctionnement de l'application, supprimer le répertoire install de votre serveur WEB.

Fenêtre de création d'un TICKET :

La fenêtre de création d'un ticket est l'interface essentielle pour enregistrer et gérer les incidents. Elle doit contenir les éléments suivants :
Faut veiller à renseigner tous les éléments nécessaires, notamment le demandeur, la catégorie, la description, la date de création, la date de résolution, le temps passé, la priorité, la criticité et l'état.

Différence entre PRIORITE et CRITICITE :

La différence entre la priorité et la criticité réside dans leur signification et leur utilisation :
Priorité : La priorité concerne la manière dont l'incident est classé en fonction de l'urgence de sa résolution. Les priorités peuvent être définies comme "Basse", "Moyenne", "Haute", etc. Elle détermine l'ordre dans lequel les tickets sont traités.
Criticité : La criticité indique à quel point l'incident a un impact sur les opérations de l'organisation. Elle peut être classée en termes de "Critique", "Majeure", "Mineure", etc. Elle permet de hiérarchiser les problèmes en fonction de leur gravité.

État d'un ticket :

Les états des tickets peuvent varier en fonction des besoins de l'organisation, mais généralement, ils incluent :
Nouveau : L'incident a été enregistré mais n'a pas encore été traité.
En cours : Le ticket est actuellement pris en charge par un technicien.
En attente : Le ticket est en attente de quelque chose, comme des informations complémentaires du demandeur.
Résolu : L'incident a été résolu et confirmé par le demandeur.
Fermé : Le ticket est terminé et ne nécessite plus de suivi.

En conclusion, l'installation de GESTSUP et de GLPI offre à notre organisation des outils pour gérer efficacement les incidents, les demandes de support et les équipements informatiques, tout en garantissant une bonne traçabilité et une gestion structurée de ces processus.

Langages de Programmation Backend :

PHP :

PHP est un langage de script côté serveur qui excelle dans le traitement des demandes web. Voici quelques aspects clés :

Interprété et Dynamique : PHP est interprété à la volée, offrant une grande flexibilité. Il est également dynamique, adapté au développement agile.

Intégration HTML : PHP peut être intégré directement dans le code HTML, facilitant la création de pages web dynamiques.

Framework Laravel : Laravel, construit en PHP, propose une architecture élégante, une gestion des migrations, et une intégration poussée de l'ORM Eloquent.

WampServer avec PHP : WampServer intègre PHP en tant que module, fournissant un environnement complet et facile à configurer.

JavaScript (Node.js) :

Node.js permet d'exécuter du code JavaScript côté serveur, offrant plusieurs avantages :

Asynchronisme : Node.js utilise un modèle asynchrone, permettant une gestion efficace des opérations non bloquantes, ce qui est particulièrement utile pour les applications en temps réel.

Événements : Le modèle d'événements de Node.js facilite la gestion des événements côté serveur.

Express.js : Un framework minimaliste pour construire des applications web et des APIs robustes avec Node.js.

Frameworks Backend :

Laravel (PHP)

Laravel est un framework PHP moderne et populaire qui offre une multitude de fonctionnalités pour le développement web.

Composer

Laravel utilise Composer pour gérer les dépendances, facilitant l'intégration de bibliothèques tierces.

Eloquent ORM

Avec Eloquent, les développeurs peuvent interagir avec la base de données en utilisant des objets PHP au lieu de requêtes SQL, améliorant la lisibilité du code.

Middleware

Les middlewares permettent de filtrer les requêtes HTTP à différentes étapes, ajoutant des couches de traitement personnalisées.

Blade Templating Engine

Blade simplifie la gestion des vues et offre une syntaxe concise pour l'écriture de templates.

Bases de Données :

MySQL

MySQL, en tant que système de gestion de base de données relationnelle, s'intègre bien avec WampServer et PHP.

Administration avec phpMyAdmin

WampServer inclut phpMyAdmin, une interface web conviviale pour administrer les bases de données MySQL.

Langage SQL

MySQL utilise SQL (Structured Query Language) pour les requêtes, ce qui facilite l'interaction avec la base de données.

Gestion des Index

Les index peuvent être utilisés pour améliorer les performances des requêtes en accélérant la recherche dans les tables.

Transactions ACID

MySQL assure la cohérence des données grâce à la prise en charge des transactions ACID (Atomicité, Cohérence, Isolation, Durabilité).

MongoDB

MongoDB est une base de données NoSQL orientée document qui offre des avantages tels que :

Structure flexible

Stockage de données sous forme de documents JSON BSON.

Évolutivité horizontale

Possibilité de faire évoluer horizontalement la base de données en ajoutant des serveurs.

Mongoose (Node.js)

Un ORM pour MongoDB, facilitant l'interaction avec la base de données.

Sécurité Backend :

Principes de Sécurité PHP

Assurer la sécurité dans le développement PHP nécessite une approche proactive et complète.

Validation des Entrées

Filtrer et valider toutes les entrées utilisateur pour éviter les attaques par injection de code. Utiliser des fonctions comme htmlspecialchars pour échapper les caractères spéciaux.

Authentification et Autorisation

Implémenter des mécanismes d'authentification robustes et définir des règles d'autorisation appropriées pour protéger les ressources sensibles.

Gestion des Dépendances

Utiliser npm audit pour identifier et résoudre les vulnérabilités dans les dépendances.

Gestion des Mots de Passe

Stocker les mots de passe de manière sécurisée en utilisant des fonctions de hachage comme password_hash et password_verify.

Middleware de Sécurité

Utiliser des middlewares tels que Helmet pour sécuriser les en-têtes HTTP et csurf pour prévenir les attaques CSRF.

Authentification avec Passport.js

Passport.js facilite l'implémentation de l'authentification.

Cryptographie avec bcrypt

Utiliser bcrypt pour hacher les mots de passe de manière sécurisée.

Sécurité des Sessions

Utiliser des mécanismes sécurisés pour gérer les sessions, tels que l'utilisation de jetons CSRF et la transmission sécurisée des cookies.

Mises à Jour Régulières

Garder le serveur Wamp, PHP, Laravel, MySQL, et toutes les dépendances à jour pour bénéficier des correctifs de sécurité.
